# 개발 노트 — 결정 사항 및 개선 후보

## 컨셉 변경 이력

### 지도 기반 → 포인트클라우드 우선으로 전환 (2026-06-23)

초기 컨셉은 지도 기반 디지털트윈이었으나, 아래 이유로 변경.

- GPS 없는 사진(실내 등)도 동등하게 처리하고 싶음
- "사진 올리면 3D됨"이 더 직관적이고 포트폴리오 임팩트 큼
- 지도 연동은 Phase 3에서 선택적으로 추가 예정

### 현실 + 가상 공간 모두 지원으로 확장 (2026-06-23)

대상을 현실 공간(여행, 실내)에서 가상 공간(게임 스크린샷)으로까지 확장.

- 마비노기·원스휴먼 등 게임 하우징 스크린샷도 포인트클라우드화 가능
- "현실이든 가상이든, 기억하고 싶은 공간을 3D로"가 핵심 스토리
- 별도 기능 추가 없이 기존 파이프라인 그대로 적용 가능

## 업로드

### Server Action body size limit

현재 20MB로 설정. 고해상도 사진(아이폰 등) 대응.
추후 개선 시 클라이언트 리사이즈(canvas) 방식으로 전환 고려.

### 업로드 전 사진 편집 기능 (개선 후보)

업로드 전 불필요한 영역 제거, 크기 조정 후 포인트클라우드 변환 요청하는 편집기 추가 예정.

- 크롭/리사이즈로 파일 크기 줄이기
- 포인트클라우드 품질 향상 (노이즈 영역 제거)
- Phase 2 포인트클라우드 연동 시 함께 구현 검토

## Auth

### Google OAuth 계정 선택 화면 강제 표시

현재 브라우저에 Google 세션이 남아있으면 계정 선택 없이 바로 로그인됨.
다중 계정 사용자를 위해 항상 계정 선택 화면을 띄우고 싶다면 아래 옵션 추가.

```typescript
// src/components/ui/Header.tsx — handleSignIn
await signIn('google', { redirectTo: '/', prompt: 'select_account' });
```
